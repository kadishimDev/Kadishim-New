# תוכנית עבודה לניהול גרסאות (Git)

הפרויקט שלך כבר מחובר ל-GiHub (או שרת מרוחק אחר) והגרסאות מנוהלות.
להלן ההוראות כיצד לדחוף שינויים ולוודא שהכל מסונכרן.

## 1. מבנה התיקיות והקבצים שמתעלמים מהם (.gitignore)
הקובץ `.gitignore` בתיקייה הראשית מגדיר אילו קבצים לא יעלו לגיט.
כרגע הוא מוגדר נכון להתעלם מ:
*   `node_modules` (ספריות כבדות)
*   `dist` (קבצי הבנייה הסופיים)
*   לוגים וקבצי מערכת

## 2. קובץ אוטומטי לדחיפת שינויים (Auto Push)
יצרתי עבורך קובץ באץ' בשם `Push to Git.bat` בתיקייה הראשית.
לחיצה כפולה עליו תבצע את הפעולות הבאות:
1.  **Add**: יוסיף את כל הקבצים ששונו או נוצרו.
2.  **Commit**: ישמור אותם עם הערה הכוללת את התאריך והשעה הנוכחיים.
3.  **Push**: ידחוף את השינויים לשרת המרוחק.

## 3. ביצוע ידני (דרך הטרמינל)
אם תרצה לבצע ידנית, הפקודות הן:

```powershell
# הוספת כל השינויים
git add .

# שמירת גרסה עם הודעה
git commit -m "עדכון אתר: הוספת עריכת נפטרים ושיפורים"

# דחיפה לשרת
git push origin main
```

## 4. פתרון בעיות נפוצות
*   **התנגשויות (Merge Conflicts)**: אם ביצעת שינויים במקביל ממקום אחר, ייתכן שתצטרך לעשות קודם `git pull`. הקובץ האוטומטי לא עושה זאת כדי לא לדרוס שינויים בטעות.
*   **קבצים גדולים**: ה-SQL המקורי (`dbexport...`) הוא גדול. מומלץ לא לשמור גיבויים ישנים בגיט אם הם לא נחוצים לפיתוח.
